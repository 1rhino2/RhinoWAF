# RhinoWAF 1.0.0

**Release date:** May 2026

First semver release. Same feature set as the pre-1.0 development tree, rebranded and hardened for production use.

## Highlights

- **Unified protection pipeline** - `ProtectRequest()` is the single gate for built-in routes and multi-vhost reverse proxy traffic (rate limits, IP rules, smuggling detection, sanitization).
- **Version 1.0.0 branding** - CLI banner, `/health`, error pages, webhooks, and proxy headers all report `1.0.0`.
- **`--version` flag** - `rhinowaf -version` prints build metadata.
- **`scripts/test-all.ps1`** - One command for tests, build, and optional lint on Windows.

## Fixes

- Multi-vhost mode no longer skips WAF checks before proxying to backends.
- HTTP smuggling detection wired into the main protection path (severity 4+ blocked by default).
- Trusted client detection uses `strings.Contains` instead of a hand-rolled substring loop.
- CI ldflags target `main.Version` / `main.BuildTime` consistently.

## Install

```bash
go build -o rhinowaf ./cmd/rhinowaf
./rhinowaf -version
```

## Verify

```powershell
powershell -File scripts/test-all.ps1
```

Or:

```bash
go test ./...
make build
```

Older internal version labels (v2.x) in `docs/CHANGELOGS/` are historical dev milestones. Public releases start at **v1.0.0**.
